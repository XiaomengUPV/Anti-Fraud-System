  IMPROVEMENT RECOMMENDATIONS
  ──────────────────────────────────────────────────────────

  Code Substitution (F1=0.069):
    → Very subtle fraud — non-covered code swapped for covered.
      LLM cannot easily detect this without coverage data.
    → Improve: add a coverage lookup table mapping non-covered
      codes to their covered substitutes.

  Upcoding (F1=0.182):
    → The LLM is flagging upcoding but fraud_type label
      mismatches (e.g. flagged as Diagnosis Mismatch).
    → Improve: refine prompt to distinguish upcoding from
      diagnosis mismatch more explicitly.

  Phantom Billing (F1=0.516):
    → LLM is too conservative — needs more specific guidance
      on what constitutes a clinically impossible combination.
    → Improve: add explicit examples to the prompt of phantom
      billing patterns (e.g. transplant tests for cold patients).

  Diagnosis Mismatch (F1=0.574):
    → High recall (finding many) but low precision (false alarms).
    → Improve: tighten the prompt to require stronger evidence
      before flagging — two-step check: is there ANY valid link?


  SUMMARY
  ──────────────────────────────────────────────────────────

  Overall F1 : 0.769  ✅ PASS
  Types met  : 5 / 9
  Types below: 4 / 9

  The system performs excellently on rule-based fraud types
  (Duplicate Billing F1=1.0, Modifier Abuse F1=1.0) and well
  on Code Padding (F1=0.877). The main gap is in contextual LLM
  fraud types — upcoding, unbundling, and code substitution —
  which would benefit from prompt refinement and more training
  examples in the few-shot context.
